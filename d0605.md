## 2. Zapier monolithic Zap 분석

사용자가 제공한 Zap Export 상세 문서(30단계+)를 HireCopilot GAS 컬럼 정의와 대조해 분석했습니다.

### 2.1 의도된 Zap 흐름 (구 설계)

```
Google Sheets 새 행 추가 (트리거)
  → 필터 4개 (이메일, GPA, 학위, 경력)
  → Paths 3갈래 (채용 / 보류 / 탈락)
  → Notion + Calendar Delay + Notion 체크박스 HITL
  → Gmail / Slack / Zoom / Claude / Google Docs
  → Sheets 최종 결과 업데이트
```

**설계 의도:** AI 1차 평가 후 자동화하되, **Notion 체크박스 + 캘린더 발표일 Delay**로 관리자 최종 결정권 보장 (Human-in-the-Loop).

### 2.2 GAS 실제 컬럼 (사용자 제공 doPost 기준)

스프레드시트 ID: `1swaf7dyRsVRxepLJAXVoPO3YRNV0aPYmcBLL4_tPnbE`

| 열 | 필드 |
|---|---|
| A | timestamp |
| B | candidate_name |
| C | candidate_email |
| D | position |
| E | degree |
| F | gpa |
| G | experience |
| H | fit_level |
| **I** | **hiring_opinion** (`추천` / `보류` / `비추천`) |

총 **19열 (A~S)**. T/U는 앱/GAS가 쓰지 않음 (구 Zap의 "최종 합격/탈락"용으로 별도 가정).

### 2.3 Zap Export vs GAS 불일치 (분석 결과)

| 항목 | Zap 문서 | 실제 GAS/앱 |
|---|---|---|
| 트리거 시트 ID | `1aJ0qI4...` | **`1swaf7dy...`** |
| 이메일 필터 | COL$D | **COL$C** |
| GPA 필터 | COL$G | **COL$F** |
| 학위 필터 | COL$H | **COL$E** |
| 경력 필터 | COL$H (학위와 동일 열 — 설정 오류) | **COL$G** |
| 채용 분기 | COL$J `"채용"` | **COL$I** `"추천"` |
| 보류 분기 | COL$I `"보류"` | **일치** |
| 탈락 분기 | COL$J `"탈락"` | **COL$I** `"비추천"` |
| Claude 입력 | E=직군, S=추천, T=대화 | D=포지션, J=추천이유, S=대화 |
| update_row | 행 번호 2·3 고固定 | find_row 결과 미사용 (버그 가능) |

**결론:** 구 monolithic Zap은 Export 그대로면 **트리거 시트·열 번호·키워드** 모두 어긋나 있어 정상 동작하기 어렵다. 이 Zap은 **돌릴 필요 없음**.

## 3. 아키텍처 방향 확정 (사용자 요구)

> **Zap은 안 돌아가도 됨. 코드로 최대한 구동.**  
> **앱 연결만 Zapier 전용 ZAP** — outbox 시트에 행 추가 → New Row 트리거 → Gmail 등.

### 3.1 새 설계 원칙

| 레이어 | 담당 | 도구 |
|---|---|---|
| 면접 + AI 평가 | Python | `app.py` + OpenAI |
| 자격 필터 + 분기 + 2차 질문 생성 | Python | `pipeline.py` |
| 시트 저장 / outbox 큐잉 | GAS | `gas/webhook_router.gs` |
| Gmail, Slack, Notion, Docs, Zoom | Zapier | outbox 탭당 **ZAP 1개** |

### 3.2 outbox 패턴

```
Python pipeline
  → POST { "target": "outbox_email", "row": [...] }
  → GAS appendRow
  → Zap: New Row in outbox_email → Gmail Send
```

복잡한 Filter/Paths/Delay/Code by Zapier는 **Python으로 이전**하거나 **미구현(후속)** 으로 분리.

## 4. 1차 구현 (세션 중반)

### 4.1 신규 파일

#### `pipeline.py`
- `check_screening()` — 이메일 `@`, GPA, 학위, 경력 검증
- `build_branch_actions()` — `hiring_opinion`별 outbox 행 생성
- `generate_followup_questions()` — 보류 시 LLM 2차 면접 질문
- `run_pipeline()` — interviews 저장 + outbox dispatch
- `post_to_gas()` — GAS POST + 302→GET 처리
- `load_pipeline_config()` — `.env` 로드

#### `gas/webhook_router.gs`
- `target` 필드로 시트 탭 라우팅
- 탭 없으면 헤더와 함께 자동 생성
- 하위 호환: `candidate_name`만 있는 JSON → `interviews` 저장

#### outbox 시트 스키마 (초版)

| 탭 | 열 |
|---|---|
| `outbox_email` | timestamp, to, subject, body, from_name |
| `outbox_slack` | timestamp, recipient, message |
| `outbox_notion` | timestamp, name, database, notes |
| `outbox_docs` | timestamp, candidate_name, candidate_email, content |
| `pipeline_log` | timestamp, candidate_name, branch, screening, detail |

### 4.2 `app.py` 변경 (1차)
- `send_to_zapier()` 제거 → `execute_pipeline()` 도입
- 면접 종료 후 `pipeline.run_pipeline()` 호출
- `pipeline_result` 세션 상태, 결과 UI
- `.env.example`에 `GAS_WEBHOOK_URL`, `ADMIN_EMAIL`, `ADMIN_SLACK_USER_ID` 추가

### 4.3 분기별 outbox 동작 (1차)

| hiring_opinion | outbox |
|---|---|
| 필터 탈락 | `pipeline_log`만 |
| **추천** | `outbox_notion` + (설정 시) Slack |
| **보류** | Slack + admin email + Notion + Docs(LLM 질문) |
| **비추천** | `outbox_email` (지원자 탈락 메일) |

## 9. 아직 미구현 (의도적 / 후속)

구 monolithic Zap에 있던 기능 중 **코드·outbox로 아직 옮기지 않은 것:**

| 기능 | 구 Zap | 현재 | 후속 제안 |
|---|---|---|---|
| Delay Until (합격자 발표일) | Calendar + Delay | 없음 | `outbox_scheduled` 탭 + cron/Zap |
| Notion 체크박스 HITL | Notion Find + 분기 | Notion **등록만** | Notion outbox + 별 워크플로 |
| 최종 합격/탈락 COL T/U | update_row | 없음 | `outbox_email` 또는 Sheets update outbox |
| Zoom API 직접 호출 | Code+Zoom | outbox → Zap | 현재 패턴 유지 권장 |
