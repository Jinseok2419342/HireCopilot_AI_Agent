Key Points

왜 이 구조인가

POINT 01

로직은 코드, 연결은 Zapier

필터·분기·LLM은 Python. Zapier는 New Row → 앱 액션만. Zap 안에 복잡한 로직 없음.

POINT 02

Zap 1개 = 앱 1개

Gmail Zap 수정이 Slack Zap과 무관. 앱 교체 시 해당 Zap만 변경.

POINT 03

테스트 가능한 파이프라인

pipeline.py 단위 테스트 40개 완료. 로직 변경 즉시 검증 가능.

POINT 04

재시작 없이 기준 변경

학점 커트라인·포지션 기준을 관리자 콘솔에서 실시간 변경. 코드 수정 불필요.
